# 第 2 章 算法原理

## 2.1 PageRank 的随机游走视角

PageRank 可以理解为一个“随机浏览者”在有向图上的长期访问概率分布。图中每个节点表示一个页面，边 `u -> v` 表示页面 `u` 上存在指向 `v` 的超链接。若浏览者当前位于节点 `u`，则有两种行为：

1. 以概率 `beta` 沿着 `u` 的一条出边继续访问；
2. 以概率 `1 - beta` 发生随机跳转，直接跳到任意一个节点。

如果记第 `t` 轮时的排名向量为 `r^{(t)}`，图共有 `N` 个节点，节点 `u` 的出度为 `L(u)`，则节点 `v` 的 PageRank 满足

\[
PR(v)=\frac{1-\beta}{N}+\beta \sum_{u \to v}\frac{PR(u)}{L(u)}.
\]

公式由两部分组成：`(1-beta)/N` 是随机跳转带来的均匀基础概率，保证任意节点都有被访问的机会；另一部分是来自入边源点的贡献，每个源点 `u` 把自己的权重均分给所有出边终点。

若把图写成转移矩阵 `M`，其中

\[
M[u,v]=
\begin{cases}
\frac{1}{L(u)}, & u \to v \\
0, & \text{otherwise}
\end{cases}
\]

则幂迭代形式可写为

\[
r^{(t+1)}=\beta M^{T}r^{(t)}+\frac{1-\beta}{N}\mathbf{1}.
\]

课程实现中采用的就是这一迭代思想：从均匀分布 `r^{(0)} = (1/N, \dots, 1/N)` 出发，不断重复更新，直到相邻两轮向量变化足够小。

## 2.2 dead-end 的成因与影响

dead-end（悬挂节点）指的是出度为 `0` 的节点。对这类节点，普通转移矩阵无法定义“把概率往哪里传出去”，因为它没有任何出边。若仍然直接套用基础公式，则这一行会全为 `0`，结果是到达 dead-end 的概率质量在下一轮迭代中被“吞掉”。随着迭代进行，排名向量的总和会持续下降，最终不再是合法的概率分布。

在实际数据集中，dead-end 并不少见——很多节点只被引用、不再向外链接。如果不处理，概率质量会逐轮泄漏，PageRank 向量总和持续下降，最终不再是合法的概率分布。而且如果团队成员各自用不同策略，实验结果就无法对齐，所以必须统一处理。

本项目冻结接口明确要求全队统一使用“质量补偿法”。设

\[
D=\{i \mid out\_deg[i]=0\},
\qquad
dangling\_mass^{(t)}=\sum_{i \in D} r_i^{(t)}.
\]

则每一轮迭代时，把这部分丢失的概率质量重新平均分配到所有节点，即

\[
r^{(t+1)}[v]
=\frac{1-\beta}{N}
+\beta \frac{dangling\_mass^{(t)}}{N}
+\beta \sum_{u \to v,\ L(u)>0}\frac{r^{(t)}[u]}{L(u)}.
\]

这样处理后，dead-end 节点的权重不会消失，而是像“随机再投放一次”那样回流到全图，向量总质量可以维持在 1 附近。

## 2.3 spider-trap 的成因与影响

spider-trap 指一个内部强连通、但几乎不向外部连边的子图。例如若一组节点之间彼此大量互链，而对外没有出口，那么随机游走一旦进入这部分区域，就会长期被困在其中。若没有随机跳转项，最终几乎全部权重都会被这类结构吸走，得到的排序会高度偏置。

spider-trap 和 dead-end 的区别在于：dead-end 导致概率无法继续传播，spider-trap 则让概率过度集中在局部封闭结构中。两者都会破坏全局随机游走分布，导致 PageRank 结果失真。

PageRank 中的 teleport 项

\[
\frac{1-\beta}{N}\mathbf{1}
\]

正是解决 spider-trap 的关键。因为无论当前处于哪个节点，浏览者都有 `1 - beta` 的概率跳出当前局部结构，回到全图范围重新采样。因此，spider-trap 不需要额外单独设一套修正规则，只要正确实现 teleport，并且 dead-end 也被统一补偿，就能保证收敛到稳定分布。

## 2.4 收敛性的理论依据

加入随机跳转后，实际迭代使用的矩阵（常称为 Google Matrix）在 `0 < beta < 1` 时满足以下性质：

1. 每一列对应的概率质量是可控的；
2. 任意节点之间都存在非零概率互达；
3. 矩阵是 primitive 的，因此存在唯一的主特征向量。

因此无论初始向量怎么选，幂迭代都会收敛到唯一的稳定分布。课程作业中固定采用

- `beta = 0.85`
- `eps = 1e-8`

`beta` 太小会稀释图结构信息，太大则对 spider-trap 更敏感、收敛更慢，`0.85` 是比较常用的折中值。`eps = 1e-8` 在精度和时间之间取得了平衡，多轮实验中 Top-100 排名保持稳定。

## 2.5 收敛判据的选择

本项目采用

\[
\lVert r^{(t+1)}-r^{(t)} \rVert_1 < \varepsilon
\]

作为收敛判据。选用 L1 范数而不是 L2 或最大范数，主要是因为 L1 直接表示全图概率质量的总移动量，物理意义比较清楚，而且对大量小幅变化也比较敏感。另外冻结接口已经统一使用 L1，三方实验对齐比较方便。

实现上也比较简单：构造一个差值缓冲区，原地做减法、取绝对值、求和就行，不需要平方或开方。

## 2.6 本章小结

综上，PageRank 的核心不是简单地“统计入边多少”，而是求一个在随机游走意义下自洽的稳态概率分布。dead-end 会造成概率质量流失，spider-trap 会造成局部质量滞留；前者必须通过质量补偿修正，后者则依赖 teleport 项被自然抑制。只要 Google Matrix 的构造正确，幂迭代就能稳定收敛到唯一解。后续第 3 章将进一步说明：在本作业的内存与时间约束下，如何把这一理论公式落实成适用于 Python 的 CSR 稀疏实现。
