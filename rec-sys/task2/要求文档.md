# Task2 Track1 要求文档

## 任务名称

Track1: 增量 SVD 矩阵分解性能优化

## 任务目标

本任务要求在已有预训练矩阵分解模型的基础上，实现一个增量更新算法。

具体来说，系统会提供：

- 用户隐向量矩阵 `P.npy`
- 物品隐向量矩阵 `Q.npy`
- 全局平均分 `global_mean.npy`
- 增量评分数据 `incremental.npy`
- 测试数据 `test.npy`

我们需要实现 `IncrementalSVD` 类，使模型能够：

1. 加载预训练的用户矩阵 `P` 和物品矩阵 `Q`
2. 使用新增评分数据进行增量更新
3. 在不重新训练全量数据的情况下提升测试集 RMSE
4. 尽可能降低 `update()` 和预测过程的总耗时

最终成绩主要由隐藏测试集上的 RMSE 改进和运行时间决定。

## 数据集说明

本任务使用 MovieLens 20M 数据集，包含约 2000 万条电影评分。

评分数据字段如下：

| 字段 | 类型 | 说明 |

| `rating` | `float` | 评分，范围为 `[0.5, 5.0]` |

## 评分预测公式

标准矩阵分解预测公式为：

```text
r_hat_ui = global_mean + dot(P[u], Q[i])
```

其中：

- `global_mean` 是全局平均分
- `P[u]` 是用户 `u` 的隐向量
- `Q[i]` 是物品 `i` 的隐向量
- 隐向量维度为 `1024`

预测结果需要截断到评分范围：

```text
0.5 <= prediction <= 5.0
```

如果遇到越界用户或越界物品，返回 `global_mean`。

## 增量更新参考思路

题面给出的 SGD 更新公式为：

```text
error = rating - prediction

P[u] = P[u] + lr * (error * Q[i] - reg * P[u])
Q[i] = Q[i] + lr * (error * P[u] - reg * Q[i])
```

可调参数包括：

- 学习率 `learning_rate`
- 正则化系数 `regularization`
- 本地训练轮数 `local_epochs`
- batch/chunk 大小
- 是否只更新用户矩阵、只更新物品矩阵，或同时更新
- 是否对误差进行裁剪

## Python 提交接口

如果选择 Python，需要提交 `solution.py`。

必须定义 `IncrementalSVD` 类，并包含以下方法：

```python
class IncrementalSVD:
    def __init__(self, num_users, num_items, latent_dim, global_mean):
        ...

    def load_base_model(self, user_matrix, item_matrix):
        ...

    def update(self, incremental_batch):
        ...

    def predict(self, user_id, item_id):
        ...
```

### `__init__`

初始化模型参数：

- `num_users`
- `num_items`
- `latent_dim`
- `global_mean`

### `load_base_model`

加载系统提供的预训练矩阵：

- `user_matrix`: 用户矩阵 `P`，形状为 `(num_users, latent_dim)`
- `item_matrix`: 物品矩阵 `Q`，形状为 `(num_items, latent_dim)`

### `update`

增量更新函数，会被评测系统计时。

输入：

```text
incremental_batch: numpy.ndarray
shape: (N, 3)
dtype: float32
每行格式: [user_id, item_id, rating]
```

注意：

- `update()` 会被多次调用
- 每次传入约 `100000` 条增量评分
- 不能重新训练全量数据
- 需要在 RMSE 和运行时间之间取平衡

### `predict`

输入：

- `user_id`
- `item_id`

输出：

- 一个 `float` 类型预测分数
- 范围必须在 `[0.5, 5.0]`

## 评测指标

主要指标为 RMSE 和运行时间。

RMSE 公式：

```text
RMSE = sqrt(mean((rating - prediction)^2))
```

有效性要求：

1. 新 RMSE 必须小于基础模型 RMSE
2. RMSE 改进幅度必须超过 `0.001`

如果 RMSE 没有有效提升，即使速度快也可能无效。

## 我们后续实现重点

Track1 是性能题，正式实现优先走 C++ 版本。

优先级：

1. 跑通官方 `solution.cpp` 和 C++ benchmark
2. 记录基础 RMSE 和时间
3. 先做低成本增量修正：
   - 统计增量数据中的用户残差
   - 统计增量数据中的物品残差
   - 用 shrinkage 防止低频用户/物品过拟合
4. 优化 `update()`：
   - 避免反复重训全部历史增量数据
   - 避免完整 1024 维 SGD 带来的巨大开销
   - 使用 OpenMP 并行统计残差
5. 优化 `predict()`：
   - 保持边界检查简单
   - 手写紧凑 dot product
   - 在基础 SVD 分数上叠加 user/item bias
6. 如果残差偏置的 RMSE 不够，再尝试少量维度的因子更新或更细的参数搜索

## 当前判断

Track1 的关键不是重新训练一个推荐模型，而是在已有 `P/Q` 的基础上：

- 用增量数据让 RMSE 有效下降
- 控制 `update()` 的计算量
- 在 RMSE 改进和运行时间之间取最优折中

因此第一版实现应以“C++ 快速有效”为主，先保证 RMSE 改进超过 `0.001`，再逐步压缩运行时间。
